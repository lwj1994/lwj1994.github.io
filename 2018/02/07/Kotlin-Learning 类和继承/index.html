继承

所有类都继承于 Any。

通过形如class A : B() 来声明继承关系。
B 带() 表示调用 B 的无参主构造。

Kotlin 中默认所有的类，属性，接口都是 final 的。open 关键字就是使其变成非 final。java 中默认都是非 final。

符合 effective java 的第 17 条：

Design and document for inheritance or else prohibit it.

需要设计继承关系的类，我们就使用 open 打开权限。否则默认都是禁止的。

如果子类没有主构造，那么所有的副构造必须通过 super() 调用父类构造。

如果子类有主构造，主构造中需要调用父类的构造，子类的副构造和一般的类一样使用 this() 调用自己的主构造。

覆写方法

由于默认的方法是 final 的，所以只能复写 open 修饰的方法。复写的方法需要用 override 修饰表示该方法是复写的。

覆写属性

和方法类似，val 属性不可以复写 var。var 可以复写 val。

覆写规则

如果一个类有多个实现关系，比如实现了多个接口，接口里面有重复的方法实现。子类需要重写这个方法。

接口和抽象类的方法和属性默认都是 open 的。
因为它们作为抽象层必须对外开放让子类访问实现。

抽象类

和 java 一样用 abstract 修饰类和方法。

可以用 abstract 重写父类的方法，不会再调用父类方法的实现。
