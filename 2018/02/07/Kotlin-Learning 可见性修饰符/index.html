Kotlin-Learning 可见性修饰符

发表于 2018-02-07 | 分类于 Android

Kotlin 中有 4 种修饰符

  1. private
  2. protect
  3. internal
  4. public

默认不写是 public。

在包中修饰

Kotlin 中，方法，属性，类，对象和接口可以直接在文件的顶级声明。 java 必须声明在类中。所以 kotlin 创建一个 .kt 文件，直接写一个 main 函数就可以运行，不需要在外面套一个类。

  1. private

    只能在当前 .kt 文件中访问

  2. protect

    不可以被顶级声明访问。顶级中不可以声明 protect

  3. internal

    在一个 module 可以随意访问

  4. public

    啥都不写就是 public。在任何地方都可以访问。

在类和接口中

  1. private

    只能在这个类内部访问

  2. protect

    和 private 一样，可以在子类中访问

  3. internal

    在 module 中可以访问到类，就可以访问到 internal 成员

  4. public

    在任何地方都可以访问。

在 Java 中，在外部类中可以访问内部类的 private 属性，而在 Kotlin 不可以，kotlin 只能在类内部访问。

在构造中

声明一个主构造的可见性。constructor 关键字不可省略

class C private constructor(a: Int) { ... }

Modules

internal 修饰符指成员的可见性只在同一个模块中可见。
